01how i think

I don't usually start with "what should I build?"

this looked much worse in the beginning

Most of what I work on starts from something personal — a problem I've lived through, or a workflow that quietly wastes everyone's time. I care less about how technically impressive something is, and more about whether it makes one specific part of someone's day less frustrating. I learn by breaking things and writing down what I notice, mostly on LinkedIn.

The first draft is usually just a question in the margin: why is this painful, and can software make it slightly less annoying?
  • campusUseful information exists, but it is usually scattered across portals, PDFs, emails, and screenshots.
  • queuesPeople can handle waiting better when the system is honest about what is happening.
  • securityNetworks work because of quiet trust assumptions. I like poking at those assumptions carefully.
  • student lifeMany student problems are not huge. They are small repeated frictions that slowly become exhausting.

01 chrome extension
personal project

Haajar (ഹാജർ)

"Students shouldn't have to make attendance decisions blindly."

Attendance portals showed a number, not a decision — so I built a privacy-first Chrome extension that reads attendance data straight from the university portal, maps it against the timetable, and tells students how many classes they can safely miss. Zero backend, zero data collection.

JavaScript Chrome APIs CSV imports

02 vehicular fog computing
Sep 2025 — Apr 2026

Fog Scheduler Using Quantum-Inspired Principles

"Why do scheduling algorithms break down under realistic vehicular workloads?"

Most scheduling papers test against neat, static assumptions — real vehicles don't move that way. I built a hybrid quantum-inspired genetic algorithm for task allocation across edge, fog, and cloud, paired with a SUMO–EdgeSimPy co-simulation for real mobility-driven workload shifts.

Python EdgeSimPy SUMO

03 real-time systems
May 2026 — present

MediQ

"Hospital waiting shouldn't feel confusing and uncertain."

Waiting is worse when you don't know where you stand. MediQ is a serverless hospital queue system covering online appointments, walk-ins, live queue movement, and ETA prediction — built around actual clinic workflows rather than a tidy demo case.

Firebase Firestore ETA prediction

04 security

DNS Spoofing Detection & Alert System

"Public wi-fi trusts a lot more than it should."

Built during my internship at Cochin International Airport, watching how much public wi-fi quietly trusts. This tool inspects DNS traffic at the packet level, flags transaction mismatches and TTL anomalies in real time, with a live dashboard and Telegram alerts.
